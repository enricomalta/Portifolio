White Papper

O Inssanos é uma plataforma web completa de gestão de bolões de loterias da Caixa desenvolvida com foco em segurança financeira, experiência do usuário e escalabilidade. Trata-se de um sistema full-stack que permite aos usuários participarem de bolões, gerenciarem suas cotas, realizarem transações financeiras seguras e interagir com um painel administrativo robusto. O projeto foi construído utilizando tecnologias modernas e implementa práticas de segurança rigorosas, essenciais para aplicações que lidam com dados sensíveis e transações monetárias.

Stack Tecnológico e Arquitetura

  • Frontend: Next.js 14 com App Router, React 18, TypeScript, Tailwind CSS e Radix UI para componentes acessíveis e consistentes.
  • Backend: APIs RESTful construídas com Next.js API Routes, utilizando TypeScript para type safety.
  • Banco de Dados: Firestore (Firebase) para armazenamento NoSQL escalável, com Firebase Admin SDK para operações server-side.
  • Autenticação e Segurança: JWT tokens armazenados em cookies HTTP-Only, bcrypt para hashing de senhas, AES-256-CBC para criptografia de dados sensíveis (CPF, dados bancários), Cloudflare Turnstile para CAPTCHA, e rate limiting com LRU Cache.
  • Pagamentos: Integração com MercadoPago para processamento de pagamentos via PIX e cartão de crédito, incluindo webhooks para confirmação automática de transações.
  • Notificações: Firebase Cloud Messaging (FCM) para notificações push em tempo real, suportando múltiplos dispositivos por usuário.
  • Testes: Jest com jsdom para testes unitários e de integração, incluindo cobertura de código.
  • Outras Tecnologias: Socket.io para comunicação em tempo real, React Hook Form com Zod para validação client-side, date-fns para manipulação de datas, e Chart.js/Recharts para visualizações de dados.

A arquitetura segue o padrão serverless (Vercel-ready), com separação clara entre client e server-side logic. O projeto utiliza hooks customizados (como useApi para requisições autenticadas com refresh automático de tokens) e contextos React para gerenciamento de estado global (autenticação, notificações, configurações).

Funcionalidades Principais

    Gestão de Usuários
    • Registro e login com validação rigorosa (email, CPF, telefone).
    • Verificação de documentos (RG/CPF) com upload de imagens e moderação administrativa.
    • Sistema de permissões baseado em roles (user, admin, moderator) com controle granular de acesso.
    • Perfil de usuário com atualização de dados, upload de avatar e histórico de transações.
    Sistema de Bolões
    • Criação e edição de bolões por administradores, com tipos variados (caixa, organizado).
    • Reserva e compra de cotas, com validação de disponibilidade e prevenção de fraudes.
    • Resultados automáticos baseados em APIs externas de loteria (como Lotofácil, Mega-Sena).

    • Depósitos via PIX e cartão de crédito através do MercadoPago.
    • Saques com validação de saldo e limites diários.
    • Carteira digital com saldo em tempo real.
    • Histórico completo de transações com filtros e exportação.

    • Dashboard com métricas em tempo real (usuários ativos, vendas, prêmios).
    • Gerenciamento de usuários: ban/unban, edição de perfis, validação de documentos.
    • Controle de bolões: criação, edição, monitoramento de vendas.
    • Sistema de suporte com tickets, respostas administrativas e notificações.
    • Relatórios e estatísticas com gráficos interativos.
    Sistema de Notificações e Comunicação
    • Notificações push via FCM para eventos importantes (ganhos, novos bolões, atualizações).
    • Sistema de suporte integrado com chat/tickets.
    • Alertas em tempo real para resultados de bolões.
    Integrações Externas
    • APIs de resultados de loteria para validação automática de prêmios.
    • MercadoPago para pagamentos seguros.
    • Firebase para autenticação, banco de dados e notificações.

Medidas de Segurança Implementadas

A segurança é o pilar fundamental do projeto, com múltiplas camadas de proteção contra ameaças comuns em sistemas financeiros:

  • Autenticação e Autorização
    • JWT tokens em cookies HTTP-Only (inacessíveis via JavaScript, prevenindo XSS).
    • Refresh tokens armazenados separadamente com expiração curta.
    • Sistema de permissões baseado em roles com verificação server-side em todas as rotas.
    • Logout automático em sessões suspeitas.
  • Proteção contra Ataques
    • Rate Limiting: Limitação de requisições por IP usando LRU Cache (ex.: 5 tentativas de login por minuto).
    • CAPTCHA: Cloudflare Turnstile integrado em formulários críticos (login, registro).
    • Sanitização de Dados: Remoção de caracteres perigosos e prevenção de XSS/SQL Injection em todas as entradas.
    • Validação Rigorosa: Uso de Zod schemas para validação server-side de todos os dados recebidos.
  • Criptografia e Proteção de Dados
    • Hashing de senhas com bcrypt (12 rounds de salt).
    • Criptografia AES-256-CBC para dados sensíveis (CPF, dados bancários) no banco de dados.
    • Timeout em operações de descriptografia para prevenir DoS.
  • Auditoria e Monitoramento
    • Audit Logs: Registro completo de todas as ações críticas (logins, transações, mudanças administrativas) com IP, timestamp e dados antes/depois.
    • Logs de moderação para ações administrativas.
    • Sistema de alertas para atividades suspeitas.
  • Segurança de Infraestrutura
    • Configuração Next.js com headers de segurança (CSP, HSTS).
    • Service Workers para cache offline seguro.
    • Validação de tokens em todas as requisições API.

  • Testes Automatizados: Cobertura de testes unitários e de integração com Jest, focando em APIs críticas.
  • Validação de Dados: Zod schemas para validação consistente em client e server.
  • Gerenciamento de Estado: Contextos React para estado global, com hooks customizados para lógica reutilizável.
  • Performance: Otimização com Next.js (SSR, ISR), lazy loading de componentes e cache inteligente.
  • Acessibilidade: Componentes Radix UI para conformidade com WCAG.
  • Documentação: Código bem comentado com JSDoc, e estrutura modular para manutenção.
  • CI/CD: Scripts npm para build, lint e testes, preparados para deployment em Vercel.
